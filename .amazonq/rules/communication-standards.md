# Communication & Copy Standards

## Language Requirements
- **Primary language**: British English spelling and grammar
- **Spelling**: Use British spellings (realise, colour, centre, organised, etc.)
- **Grammar**: Follow British conventions for punctuation and sentence structure
- **Date format**: DD/MM/YYYY or "24th December 2025" format

## Brand Voice & Tone
- **Direct and honest**: No corporate doublespeak or marketing fluff
- **Human and conversational**: Write like explaining to a mate, not a customer
- **Technically accurate**: Precise without being condescending
- **Privacy-focused**: Emphasise user control and data protection

## Copy Guidelines

### Error Messages
- **Be specific**: "SMTP server rejected your password" not "Authentication failed"
- **Be helpful**: Include next steps or links to documentation
- **Be human**: "Something's gone wrong" is better than "An error occurred"
- **British tone**: "Bloody hell, that didn't work" over "We apologise for the inconvenience"

### User Interface Text
- **Button labels**: "Create alias" not "Generate forwarding mechanism"
- **Form labels**: Clear, direct language without jargon
- **Help text**: Explain what happens, not what the system does
- **Success messages**: Confirm what the user accomplished

### Email Communications (Porter Persona)
- **Tone**: Professional but approachable digital assistant
- **Style**: Helpful, security-conscious, no-nonsense
- **Language**: British English with technical accuracy
- **Format**: Plain text preferred, clear structure

### API Documentation
- **Descriptions**: Explain purpose and expected behaviour
- **Examples**: Real-world scenarios, not abstract concepts
- **Error codes**: Clear explanations of what went wrong and how to fix it

## Forbidden Phrases
- "We apologise for any inconvenience" → "This is broken, we're fixing it"
- "Please be advised" → Just say what you need to say
- "At this time" → "Currently" or just omit
- "Going forward" → "From now on" or omit
- "Leverage" → "Use"
- "Solutions" → Specific feature names

## Required Phrases
- Use "email alias" not "forwarding proxy"
- Use "passkey" not "WebAuthn credential"
- Use "privacy" not "data protection" in user-facing text
- Use "sign in" not "log in" or "authenticate"

## Technical Writing
- **Code comments**: Explain why, not what
- **Commit messages**: Imperative mood, British spelling
- **Documentation**: Task-oriented, assume intelligence but not expertise
- **Variable names**: Clear, British spelling where applicable
